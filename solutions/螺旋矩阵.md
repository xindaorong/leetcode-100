​        可以通过模拟螺旋矩阵的路径。初始位置是矩阵的左上角，初始方向是**向右**，当路径超出界限或者进入之前访问过的位置时，**顺时针**旋转，进入下一个方向。

判断路径是否进入之前**访问过的位置**需要使用一个与**输入矩阵大小相同的辅助矩阵 visited**，其中的每个元素表示该位置是否被访问过。当一个元素被访问时，将 visited 中的对应位置的元素设为已访问。

如何判断路径是否结束？由于矩阵中的每个元素都被访问一次，因此路径的长度即为矩阵中的元素数量，当路径的长度达到矩阵中的元素数量时即为完整路径，将该路径返回。

​      补充

这个条件判断用于检测下一个位置是否**无法继续前进**，共对应以下三种情况：

1. **上边界越界**：`nextRow < 0`，即下一个位置的行索引小于 0，超出了矩阵顶部。

2. **下边界越界**：`nextRow >= rows`，即下一个位置的行索引大于等于行数，超出了矩阵底部。

3. **左边界越界**：`nextColumn < 0`，即下一个位置的列索引小于 0，超出了矩阵左边界。

4. **右边界越界**：`nextColumn >= columns`，即下一个位置的列索引大于等于列数，超出了矩阵右边界。

5. **已访问过**：`visited[nextRow][nextColumn]` 为真，即下一个位置在之前的遍历中已经被放入结果中，说明当前方向遇到了已经走过的路径。

   

   **上边界越界**：`nextRow < 0`，即下一个位置的行索引小于 0，超出了矩阵顶部。

​    考虑一个**只有一行**的矩阵（`rows = 1, columns = n`）：

1. 初始方向向右，遍历整行，到达最右侧。
2. 下一个位置（右边界外）触发“右边界越界”，方向转向下。
3. 向下移动时，`nextRow = 0 + 1 = 1 >= rows`，触发“下边界越界”，方向转向左。
4. 向左移动时，`nextColumn` 不断递减，直到 `column = 0` 时，再左移一步 `nextColumn = -1`，触发“左边界越界”，方向转向上。
5. 现在方向向上，此时 `row = 0`，`nextRow = -1`，**“上边界越界”条件成立**。

虽然在这个例子中，当方向转为向上时，循环已经执行了 `total` 次（所有元素已收集完毕），不再进行下一次移动，但条件判断本身依然会被执行。如果矩阵是**单列**的情况，对称地也会出现“左/右边界越界”被提前触发，但向上/向下边界检查仍然在代码中保留。
